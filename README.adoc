= kvf - Key Value File CLI tool

Super simple file based Key-Value command line tool that works with concurrent R/W operations.

== Installation

=== Quick Install (Recommended)

One-liner that auto-detects your OS and architecture:

==== Using curl

[source, bash]
----
curl -sL https://raw.githubusercontent.com/oxio/kvf/main/install.sh | bash
----

==== Using wget

----
wget -qO- https://raw.githubusercontent.com/oxio/kvf/main/install.sh | bash
----

=== Go Install

If you have Go installed:

[source, bash]
----
go install github.com/oxio/kvf@latest
----

=== Manual Download

Download the appropriate binary for your platform from the https://github.com/oxio/kvf/releases/latest[latest release].

==== Linux

[source, bash]
----
# amd64 (x86_64)
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-linux-amd64 -o kvf
chmod +x kvf
sudo mv kvf /usr/local/bin/

# arm64 (aarch64)
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-linux-arm64 -o kvf
chmod +x kvf
sudo mv kvf /usr/local/bin/

# arm (32-bit, armv7)
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-linux-arm -o kvf
chmod +x kvf
sudo mv kvf /usr/local/bin/
----

==== macOS

[source, bash]
----
# Apple Silicon (M1/M2/M3/M4/M5)
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-darwin-arm64 -o kvf
chmod +x kvf
sudo mv kvf /usr/local/bin/

# Intel (x86_64)
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-darwin-amd64 -o kvf
chmod +x kvf
sudo mv kvf /usr/local/bin/
----

==== Windows

[source, powershell]
----
# amd64 (x86_64)
Invoke-WebRequest -Uri "https://github.com/oxio/kvf/releases/latest/download/kvf-windows-amd64.exe" -OutFile "kvf.exe"

# arm64
Invoke-WebRequest -Uri "https://github.com/oxio/kvf/releases/latest/download/kvf-windows-arm64.exe" -OutFile "kvf.exe"
----

Or using PowerShell with `curl` alias:

[source, powershell]
----
curl -sL https://github.com/oxio/kvf/releases/latest/download/kvf-windows-amd64.exe -o kvf.exe
----

== Usage

[source, bash]
----
kvf set /path/to/file.kv KEY_ONE value
kvf set /path/to/file.kv KEY_TWO "value with spaces"

kvf get /path/to/file.kv KEY_ONE
kvf get /path/to/file.kv KEY_TWO
----

.Output
----
value⏎
value with spaces⏎
----

NOTE: The keys are case-sensitive.

=== Getting values from multiple files

The value from last file in which the key is found will be returned.

.1. Set different values for two files
[source, bash]
----
kvf set .env       APP_ENV prod
kvf set .env.local APP_ENV dev
----

.2. Get the value from first and second file in that order
[source, bash]
----
kvf get .env .env.local APP_ENV
----

.Output
----
dev⏎
----

=== Setting values for multiple files

.1. Set the value for both files
[source, bash]
----
kvf set .env .env.local APP_PORT 8000
----


.2. Get from first file
[source, bash]
----
kvf get .env APP_PORT
----

.Output
----
8000
----

.3. Get the value from second file only
[source, bash]
----
kvf get .env.local APP_PORT
----

.Output
----
8000
----


== Syntax

The syntax of the supported file is mostly compatible with `.env` files syntax.

----
# Comments
key=value
key = value
key="value"
key='value'

# Multi-line values (both quote types supported)
PRIVATE_KEY="-----BEGIN KEY-----
MIIEpAIBAAKCAQEA0Z
-----END KEY-----"

MESSAGE='Hello
World!'
----

=== Multi-line Values

Multi-line values are supported using both double and single quotes. The opening quote starts the value, and the matching closing quote ends it. All content between, including newlines, is preserved.

To include a literal quote character inside a multi-line value, escape it with a backslash:

----
KEY="This has an \"escaped\" quote inside"
----

== Features

* [*] Support for single and double quotes in values
* [*] Support for simple full-line comments (line must start with `#` character)
* [*] Concurrent R/W operations
* [*] Reading multiple files with single command
* [*] Support for multi-line values
